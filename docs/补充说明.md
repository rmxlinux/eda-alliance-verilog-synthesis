## 这些补充解决的核心问题

### 问题 1：为什么要生成 VBE？
**回答**：VBE 是 Alliance 后端的标准输入接口，只要生成合法 VBE，就能无缝接入综合链，复用成熟的 BOOM/BOOG/LOON。

### 问题 2：VBE 能表达什么？
**回答**：VBE 可以表达组合逻辑（通过连续赋值）、时序逻辑（通过 REGISTER + GUARDED BLOCK）、向量运算，足以表达可综合的 Verilog 子集。

### 问题 3：Verilog 和 VBE 有什么区别？
**回答**：主要是语法差异（索引方向、关键字）和表达方式差异（过程块 vs 连续赋值），需要通过 AST + lowering 来转换。

### 问题 4：为什么不直接用 Alliance 的 BEH API？
**回答**：为了快速闭环和独立开发，选择文本生成更简单、更容易调试，未来可以改进为 API 方式。

### 问题 5：转换是否可行？
**回答**：通过四个实例展示了从简单到复杂的转换过程，证明了可行性。
